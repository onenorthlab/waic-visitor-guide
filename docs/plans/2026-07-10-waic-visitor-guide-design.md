# WAIC 2026 访客获得感导航站设计

## 1. 产品定义

这是一个面向 WAIC 2026 现场访客的独立参访决策工具。它不替代官网、Hi WAIC APP、票务或论坛预约，而是把用户提供的 175 场完整论坛日程与 WAIC 官方访客信息转化成可操作的问题答案：什么值得去、为什么适合我、是否撞期、换馆是否现实、空档时间该如何使用。

北极星不是一个不可解释的总分，而是：

`高价值体验 / 有效参访小时 = 用户标记为值得的已参加活动数 / 实际在场小时`

首版无法预知用户最终评价，因此使用四个可解释的前置指标：目标相关赛道覆盖率、有效内容时长、预计换馆次数、可避免的移动与等待损耗。站点必须清楚区分来源事实和规划建议。

设计判读：这是面向大型国际科技大会访客的编辑型数据产品，语言应官方、未来、可信，借用 Visual Capitalist 的信息叙事语法，但现场使用优先于长图阅读。

设计参数：

- `DESIGN_VARIANCE: 8`，使用不对称首屏、主题地形图和跨视图联动。
- `MOTION_INTENSITY: 6`，动效只表达数据关系、状态变化和交互反馈。
- `VISUAL_DENSITY: 7`，信息密度高，但移动端收敛成单列决策卡。

## 3. 方案比较与选择

### 方案 A：纯日程仪表盘

优点是实现快、筛选清楚。缺点是容易像企业后台，无法先帮助用户理解 175 场活动的总体结构，也缺少符合用户参考站的创意表达。

### 方案 B：数据故事加路线工作台

先用一天一问的方式解释大会机会结构，再把图表选择带入完整日程筛选。主题、时间和场馆图表共用 `ExplorerSelection`；路线工作台使用独立、可分享的 `PlannerState`，避免探索性点击悄然改写用户路线。优点是兼顾记忆点和现场可用性。首版选择此方案。

### 方案 C：地图优先导览

空间感强，但精确路线依赖地图坐标、实时交通和容量数据。官方目前只公开接驳频率，没有稳定的实时接口。首版只做馆区示意与保守缓冲，不伪装成实时导航。

## 4. 信息架构

单页 SPA，导航保持一行：`总览 / 机会全景 / 路线工作台 / 全部日程 / 场馆指南`。

### 首屏：175 场，只走最值得的一条线

左侧是价值主张与两个行动：开始规划、查看全景。右侧使用项目生成的 `public/assets/waic-data-terrain.webp`，它把四天、七类场馆与活动节点抽象成汇聚路线。图片没有文字或官方标识，避免冒充 WAIC 官方主视觉。

首屏下方只显示三条真实洞察：

- 74% 的论坛集中在 7 月 18-19 日。
- 64% 的论坛来自综合、大模型与产业智能三大赛道。
- 52% 的论坛位于世博中心。

### 机会全景

用联动视图回答三个问题：哪天最密、哪个主题最拥挤、哪个馆最集中。主视图是日期与时段热力图，辅视图是 13 赛道主题星群和 7 类场馆分布。点击任何数据单元都会更新日程筛选，并提供静态文字列表作为无动画和无视觉依赖兜底。

### 30 秒获得感罗盘

用户选择身份、最多两个目标、日期、主题兴趣与参访节奏。默认身份模板只提供起点，用户可以继续编辑。首版身份包括技术研发、企业决策者、创业者、投资人、研究者、内容创作者和首次参会者。

### 路线工作台

结果由时间轴、场馆示意和注意力预算共同表达。每个推荐场次必须显示推荐理由、时间与馆区；结果顶部统一显示醒目的入场资格声明与官方注册链接。用户可以移除活动并触发下一最佳组合重算，也可以从完整日程重新加入，工作台立即重算冲突、换馆缓冲和空档。

### 全部日程

支持中文和英文搜索、日期、赛道、场馆筛选。默认只展示最相关的前 24 条，用户主动选择“查看全部”才展开完整列表。活动详情在底部抽屉中呈现，包含官方搜索/注册入口、高德搜索链接和加入行程按钮。

### 场馆指南

使用七类馆区节点和四个核心场馆官方定位。接驳线路只显示官方公布的发车频率。任何规划时间都标为“建议缓冲”，不标为官方车程。

## 5. 数据模型与规范化

```ts
type Event = {
  id: number;
  category: CategoryId;
  titleZh: string;
  titleEn?: string;
  date: "2026-07-17" | "2026-07-18" | "2026-07-19" | "2026-07-20";
  startTime: string;
  endTime: string;
  startMinutes: number;
  endMinutes: number;
  locationZh: string;
  locationEn?: string;
  venueGroup: VenueGroupId;
  venueDetailId: string;
  searchText: string;
  source: "user-supplied-waic-schedule";
};
```

规范化规则：

1. 双行字段以首个换行拆分中文和英文。没有第二行时保留单语，不猜译。
2. 时间只从中文首行读取 `HH:mm-HH:mm`。所有 175 行必须可解析且结束晚于开始。
3. `西岸国际会展中心` 归一为官方命名 `徐汇西岸国际会展中心`，UI 可使用较短显示名。
4. `其他场馆` 不能合并为一个可导航位置，必须用具体地址生成独立 `venueDetailId`。
5. 搜索文本使用中英文标题、分类、地点和场馆拼接并转小写。
6. 源数据不含演讲人、展位、容量和实时状态，因此这些字段首版不存在。

## 6. 可解释路线算法

算法目标不是选“全场最好”，而是在用户约束内生成一组无冲突、少折返、主题相关且有适度多样性的活动。

### 事件匹配分

每个活动的内部匹配值只用于排序，UI 显示文字理由而非虚构精确评分：

- 用户直接选择该赛道：基础权重 5。
- 身份模板对应赛道：权重 3。
- 活动赛道命中用户目标的预设关联：使用低于身份模板的辅助权重。界面明确标为“目标相关赛道”，不把它表述成标题语义理解或精确目标完成度。
- 与已选路线形成新赛道：多样性加 1。
- 同馆连续：效率加 1。

### 换馆建议缓冲

规划器使用保守、可配置的建议值：

- 同一房间或同一详细场地：10 分钟。
- 同一馆区不同房间：15 分钟。
- 世博中心与世博展览馆：25 分钟。
- 官方接驳覆盖的核心馆区切换：45 分钟。
- 酒店与相邻世博馆区：30 分钟。
- 张江、临港或其他分散场地：60 分钟。

这些数值是规划建议，不是官方交通时长。场馆指南另行展示官方的 10 或 15 分钟发车频率。

### 选择方法

对每个日期使用带换馆缓冲的加权区间调度：按结束时间排序，动态规划计算最大总匹配值，并在分值相同的情况下依次偏好更少换馆、更多目标相关赛道覆盖和更长有效内容时间。参访节奏控制每天最多 2、3 或 4 个活动。

路线结果展示：

- 活动数量与有效内容时长。
- 目标相关赛道覆盖数量。
- 赛道多样性。
- 换馆次数。
- 建议移动与缓冲时间。
- 如果存在，展示不能同时参加的高相关候选及替代理由；如果不存在，明确说明当前约束下没有额外高相关冲突项，不虚构候选。

## 7. 视觉系统

页面采用自动明暗模式，但整页在任一时刻保持同一主题，不在章节间反转。

核心 tokens：

```css
:root {
  --ink: #0a1533;
  --page: #f8f9fa;
  --surface: #ffffff;
  --canvas-blue: #eff3ff;
  --muted: rgba(10, 21, 51, 0.62);
  --line: #ebe7ff;
  --accent: #5769ff;
  --accent-soft: rgba(87, 105, 255, 0.14);
  --radius-panel: 24px;
  --radius-control: 14px;
  --radius-pill: 999px;
}
```

蓝紫品牌渐变只用于首屏资产、激活状态和少量关键数字。橙、玫红、青与绿只在数据节点中表达分类，不做大面积背景。当前字体栈使用设备自带的 `SF Pro Display / SF Pro Text / PingFang SC / Microsoft YaHei / system-ui`；未持有可确认授权的 MiSans 文件，因此不热链或打包第三方字体资产。

形状规则：面板 24px，输入和普通按钮 14px，标签与小型切换全圆角。阴影只用于真正抬高的浮层和活动详情抽屉。

## 8. 动效、无障碍与状态

- 首屏数据地形做轻微视差，表达路线汇聚；降低动态偏好时完全静止。
- 热力图和主题星群只动画 `transform` 与 `opacity`，点击后使用布局过渡解释筛选变化。
- 不使用 `window` scroll 监听，不做滚动劫持。
- 所有图表都有表格或文本替代，颜色之外再用标签和数值编码。
- 支持键盘导航、可见焦点、抽屉焦点锁定、Escape 关闭、44px 移动端触控目标。
- 数据加载采用与最终形状一致的骨架；无结果提供清除筛选；数据异常显示来源说明和重试动作。
- 用户选择保存在 URL 和 localStorage，不需要登录、定位、通讯录或联系人权限。

## 9. 技术架构

- React 19 + Vite + TypeScript。
- Tailwind CSS v4 负责布局与响应式，CSS variables 负责品牌 tokens。
- Motion 负责状态和数据关系动效。
- Phosphor Icons 作为唯一图标家族。
- 图表优先使用语义 HTML、CSS Grid 和小型数据 SVG，不引入重量级地图 SDK。
- Vitest + Testing Library 验证数据、规划器和关键交互。
- Playwright CLI 验证桌面、移动端、分享恢复、空结果和 reduced motion，不在项目中维护额外的浏览器测试框架。
- Cloudflare Workers Static Assets，独立 Worker `waic-visitor-guide`，不绑定 D1 或 R2。

## 10. 部署与信任边界

首发部署到 `waic-visitor-guide.ingle.workers.dev`。如果当前账号可管理 `waytoagi.com`，再绑定 `waic.waytoagi.com`；不能为了自定义域名影响首发。

页面页脚固定展示：

`WaytoAGI 制作的独立访客规划工具。活动时间与入场规则请以 WAIC 官网和 Hi WAIC APP 最新发布为准。`

所有预约和票务跳回官方注册页。官方 Logo、照片和主视觉不作为项目内持久资产，避免未确认授权时热链或暗示官方合作。

## 11. 成功标准

1. 175 场活动全部可检索，日期、分类和场馆统计与源工作簿一致。
2. 规划器不会生成时间重叠或小于建议换馆缓冲的路线。
3. 用户 30 秒内可生成第一条路线，并能看懂每个推荐理由。
4. 手机 390px 宽度下首屏、规划器、路线和列表无横向溢出。
5. 所有核心交互可键盘完成，reduced motion 下没有自动动画。
6. `test / typecheck / build / data:check` 与 Playwright CLI 走查全部通过。
7. Cloudflare 线上路由返回 200，桌面与移动浏览器走查通过。
